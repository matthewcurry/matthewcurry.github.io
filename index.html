Matthew L. Curry, Ph.D., is a Senior Member of Technical Staff in the Scalable System Software department at Sandia National Laboratories. He first gained notoriety for creating one of the first systems-level applications for GPU computing, Gibraltar: A software RAID system that offloads parity computation. He has since worked on parallel file system design via Sirocco, a peer-to-peer-inspired parallel object storage system. He also works on storage system characterization and reliability modeling.

Interests

This is list of technologies I have done meaningful work with in the past, in no meaningful order. I'm always interested in talking about these things, even if only maybe slightly related.

  • GPU computing, both with OpenGL and CUDA
  • Erasure coding, particularly RAID 6 schemes, k+m schemes, Reed-Solomon coding
  • RAID system design, including hardware, software, and distributed
  • Distributed object stores, including RADOS
  • Distributed and parallel file systems, including Ceph and Lustre
  • Distributed storage system design
  • Application I/O patterns and optimization
  • Behavioral modeling of disks and disk systems
  • Reliability modeling of large-scale storage systems
  • Linux kernel internals
